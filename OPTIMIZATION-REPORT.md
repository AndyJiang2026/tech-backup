## ✅ 优化详情

### 1️⃣ 技能精简

**移除的技能**:
| 技能 | 原因 | 替代方案 |
|------|------|---------|
| `summarize-pro` | 与 `summarize` 功能重叠 | 使用 `summarize` 统一处理 |
| `minimax-speech` | 与 `minimax-tts-cn` 功能相似 | 合并使用场景，按需选择 |
| `minimax-tts-cn` | 与 `minimax-speech` 功能相似 | 合并使用场景，按需选择 |

**影响**: 
- 减少技能维护成本
- 降低 Agent 路由决策复杂度
- 无功能损失（功能已合并）

### 3️⃣ 性能调优

**搜索引擎优化**:
```
优先使用：searxng（隐私保护、无追踪）
备用：minimax-mcp（网络搜索 + 图像理解）
```

**API 用量监控**:
- 启用 `minimax-usage` 监控
- 配置告警阈值（待 API Key 配置完成后设置）
- 建议阈值：日用量 80% 告警，95% 紧急告警

**TTS 优化**:
- 合并 `minimax-speech` 和 `minimax-tts-cn` 使用场景
- 中文内容优先使用 `minimax-tts-cn`
- 多语言内容使用 `minimax-speech`

---

### 4️⃣ 流程优化

**任务路由优化**:
| 优化项 | 优化前 | 优化后 |
|-------|-------|-------|
| 网络搜索 | 小马 + 研究分析 | 小马/研究分析（灵活调度） |
| 内容摘要 | summarize + summarize-pro | summarize（统一入口） |
| API 监控 | 无 | minimax-usage（小马直管） |

**路由原则**:
1. 简单任务 → 小马直接处理（减少路由层级）
2. 专业任务 → 路由到对应专业 Agent
3. 深度研究 → 研究分析 Agent 独立处理
