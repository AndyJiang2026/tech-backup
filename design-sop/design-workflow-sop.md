# 平面设计工作流程 SOP

**版本**: v1.0  
**创建日期**: 2026-04-17  
**适用**: 所有 AI 设计任务

---

## 📋 标准工作流程

```
需求分析 (5m) → 风格定位 (3m) → Prompt 构建 (5m) → 
AI 生成 (30s) → 质量检查 (3m) → 交付输出 (2m)

总计：约 15-20 分钟
```

---

## 1️⃣ 需求分析（5 分钟）

### 1.1 需求收集

**使用模板**：
```markdown
## 设计需求表

**设计类型**: ⬜海报 ⬜Logo ⬜插画 ⬜UI ⬜其他
**使用场景**: ⬜线上 ⬜线下 ⬜印刷 ⬜社交媒体
**目标受众**: 
**核心信息**: 
**尺寸要求**: 
**截止时间**: 
**参考案例**: 
```

### 1.2 需求确认

**确认话术**：
```
确认一下您的需求：
- 设计类型：[类型]
- 使用场景：[场景]
- 核心信息：[信息]
- 风格偏好：[风格]
- 交付时间：[时间]

确认无误后我开始设计。
```

### 1.3 输出物

- ✅ 设计需求表（已填写）
- ✅ 用户确认记录

---

## 2️⃣ 风格定位（3 分钟）

### 2.1 参考收集

**操作**：
1. 从 Prompt 库选择对应类别
2. 查看 3-5 个相关 Prompt
3. 选择最匹配的 1-2 个

### 2.2 色彩方案

**从配色库选择**：
- 红色教育：中国红 + 金色
- 科技未来：蓝紫 + 银色
- 清新自然：绿色 + 蓝色
- 高端奢华：黑色 + 金色

### 2.3 字体选择（如需后期添加文字）

**推荐搭配**：
- 标题：书法体/粗体
- 副标题：现代无衬线体
- 正文：细黑体

### 2.4 输出物

- ✅ 风格参考（1-2 个）
- ✅ 色彩方案（1 个）
- ✅ 字体方案（可选）

---

## 3️⃣ Prompt 构建（5 分钟）

### 3.1 使用模板

**基础结构**：
```
[主题] + [风格关键词] + [色彩方案] + [核心元素] + 
[构图方式] + [技术参数]
```

### 3.2 Prompt 优化技巧

**有效关键词**：
- ✅ "电影级质感" (+20% 质感)
- ✅ "专业摄影" (+15% 真实感)
- ✅ "对称式构图" (+25% 稳定性)
- ✅ "高清" (+10% 清晰度)

**避免的词汇**：
- ❌ "完美"（过于抽象）
- ❌ "最好"（AI 无法理解）
- ❌ "超高清"（与"高清"效果相近）

### 3.3 Prompt 检查清单

- [ ] 主体描述清晰
- [ ] 风格关键词准确
- [ ] 色彩方案明确
- [ ] 构图方式指定
- [ ] 技术参数完整

### 3.4 输出物

- ✅ 最终 Prompt（已优化）

---

## 4️⃣ AI 生成（30 秒）

### 4.1 调用 API

**通义万相配置**：
```bash
模型：wanx-v1
尺寸：1024*1024 (方版) / 768*1152 (竖版)
异步：X-DashScope-Async: enable
轮询：3 秒间隔，最多 15 次
```

### 4.2 生成策略

**多版本策略**：
- 生成 1 版初稿（快速验证）
- 如不满意，调整 Prompt 再生成 1-2 版
- 选择最佳版本

### 4.3 输出物

- ✅ 生成图片（1-3 版）
- ✅ 图片 URL（已保存）

---

## 5️⃣ 质量检查（3 分钟）

### 5.1 快速检查（30 秒）

- [ ] 主体清晰可识别
- [ ] 色彩搭配和谐
- [ ] 无明显 AI 瑕疵
- [ ] 符合设计主题

### 5.2 详细检查（2 分钟）

**使用检查表**：`design-checklist/quality-checklist.md`

**必查项**：
- [ ] 基础检查（分辨率/格式/大小）
- [ ] 视觉检查（主体/色彩/构图）
- [ ] 内容检查（信息/错字/版权）

### 5.3 评分（1 分钟）

**评分标准**：
- 90-100 分：优秀，直接交付
- 80-89 分：良好，小修后交付
- 70-79 分：合格，需要优化
- <70 分：不合格，重新设计

### 5.4 输出物

- ✅ 质量检查表（已填写）
- ✅ 综合评分
- ✅ 质量等级

---

## 6️⃣ 交付输出（2 分钟）

### 6.1 交付清单

**标准交付**：
- [ ] 成品图（PNG/JPG）
- [ ] 设计说明（Markdown）
- [ ] 使用建议
- [ ] 源文件（如可编辑）

### 6.2 文件命名

**规范**：
```
[项目名]_[类型]_[版本]_[日期]_[尺寸].[格式]

示例:
longzheng_poster_v1_20260417_1024x1024.png
```

```

**设计说明**:

- 色彩方案：[颜色]
- 使用场景：[场景]

**交付文件**:
- [文件 1]
- [文件 2]

**使用建议**:
- [建议 1]
- [建议 2]

如需调整请告诉我！
```

### 6.4 输出物

- ✅ 交付文件包
- ✅ 交付说明

## 🔧 常见问题处理

### Q1: 生成效果不理想？

**处理流程**：
1. 分析具体问题（色彩/构图/元素）
2. 优化 Prompt（添加具体描述）
3. 重新生成（1-2 版）
4. 如仍不理想，更换风格

### Q2: AI 生成文字错误？

**处理方案**：
- AI 文字能力弱，需后期添加
- 使用 Photoshop/Canva 添加文字
- 提供文字排版建议

### Q3: 尺寸不符合要求？

**解决方案**：
- 通义万相支持 4 种尺寸
- 选择最接近的尺寸
- 后期裁剪或放大

### Q4: API 限流？

**处理方法**：
- 等待 1-2 分钟后重试
- 降低生成频率（5 秒间隔）
- 使用备用 API（MiniMax）
