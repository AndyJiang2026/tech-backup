## 二、个人信息保护 (PIPL 合规)

### 2.1 适用场景

| 业务场景 | 收集信息类型 | 法律依据 | 合规要点 |
|----------|--------------|----------|----------|
| 用户注册 | 手机号、邮箱、密码 | 合同履行 | 明示告知、单独同意 |
| 实名认证 | 姓名、身份证号 | 法定义务 | 单独同意、加密存储 |
| 支付结算 | 银行卡号、支付账号 | 合同履行 | 加密传输、不存储 CVV |
| 行为分析 | 浏览记录、点击行为 | 合法利益 | 去标识化、可退出 |
| 个性化推荐 | 兴趣偏好、使用习惯 | 单独同意 | 提供关闭选项 |
| 设备信息 | IP 地址、设备 ID | 安全保障 | 最小必要、定期删除 |

#### 2.2.2 单独同意情形

以下情形需获取用户**单独同意**（不能一揽子同意）：

| 情形 | 同意方式 | 记录要求 |
|------|----------|----------|
| 敏感个人信息处理 | 勾选框/弹窗 | 单独记录 |
| 个人信息出境 | 弹窗 + 确认 | 单独记录 |
| 向第三方提供 | 勾选框 | 单独记录 |
| 公开披露 | 弹窗 + 确认 | 单独记录 |
| 变更使用目的 | 弹窗 + 确认 | 单独记录 |
| 保存期限变更 | 弹窗 + 确认 | 单独记录 |

### 2.3 敏感个人信息

**定义**：一旦泄露可能导致人格尊严或人身、财产安全受到危害的信息

| 类型 | 具体信息 | 处理要求 |
|------|----------|----------|
| 生物识别 | 人脸、指纹、声纹 | 单独同意 + 必要性说明 |
| 医疗健康 | 病历、体检报告 | 单独同意 + 严格保密 |
| 金融账户 | 银行卡、支付账号 | 单独同意 + 加密存储 |
| 行踪轨迹 | GPS 定位、行程信息 | 单独同意 + 最小必要 |
| 不满 14 周岁 | 儿童个人信息 | 监护人同意 + 专门规则 |

## 三、数据分类分级

### 3.1 数据分类

```
【数据分类框架】

一级分类          二级分类              示例
├── 用户数据       ├── 身份信息         姓名、手机号、身份证号
│                 ├── 账户信息         账号、密码、登录记录
│                 └── 行为数据         浏览记录、操作日志
│
├── 业务数据       ├── 交易数据         订单、支付、发票
│                 ├── 内容数据         用户上传内容、生成内容
│                 └── 交互数据         聊天记录、反馈信息
│
├── 技术数据       ├── 系统数据         配置、日志、监控
│                 ├── 代码数据         源代码、算法模型
│                 └── 安全数据         密钥、证书、漏洞
│
└── 管理数据       ├── 人事数据         员工信息、薪资
                  ├── 财务数据         报表、账目、预算
                  └── 经营数据         战略、计划、合同
```

### 3.2 数据分级

| 级别 | 定义 | 影响程度 | 示例 | 保护要求 |
|------|------|----------|------|----------|
| **核心数据** | 关系国家安全、国民经济命脉 | 特别严重 | 国家秘密、核心算法 | 最高级保护、境内存储 |
| **重要数据** | 危害国家安全、公共利益 | 严重 | 大规模个人信息、关键基础设施 | 严格保护、原则上境内 |
| **一般数据** | 危害个人、组织合法权益 | 一般 | 普通个人信息、业务数据 | 标准保护、可出境 |

### 3.3 公司数据分级清单

| 数据类型 | 具体数据 | 级别 | 存储要求 | 出境要求 |
|----------|----------|------|----------|----------|
| 用户身份信息 | 姓名、手机号 | 一般 | 加密存储 | 安全评估 |
| 用户生物信息 | 人脸、声纹 | 重要 | 加密 + 隔离 | 禁止 |
| 支付信息 | 银行卡号 | 重要 | 加密 + 脱敏 | 禁止 |
| 行为数据 | 浏览记录 | 一般 | 去标识化 | 安全评估 |
| 核心算法 | AIGC 模型 | 核心 | 加密 + 访问控制 | 禁止 |
| 源代码 | 平台源码 | 核心 | 加密 + 版本控制 | 禁止 |
| 财务数据 | 财务报表 | 重要 | 加密 + 权限 | 安全评估 |
| 员工信息 | 人事档案 | 一般 | 加密存储 | 安全评估 |

## 六、数据出境管理

### 6.1 出境情形

| 出境场景 | 是否出境 | 合规要求 |
|----------|----------|----------|
| 境外用户访问境内系统 | 否 | 无需额外手续 |
| 数据存储在境外服务器 | 是 | 安全评估/认证/合同 |
| 境外总部访问境内数据 | 是 | 安全评估/认证/合同 |
| 使用境外云服务 | 是 | 安全评估/认证/合同 |
| 向境外第三方提供数据 | 是 | 安全评估/认证/合同 |
| 境外机构远程运维 | 是 | 安全评估/审批 |

### 6.2 出境路径

```
【数据出境合规路径】

路径一：安全评估（重要数据/大规模个人信息）
├── 适用：重要数据、100 万 + 个人信息、10 万 + 敏感信息
├── 流程：申报→评估→批准
├── 时限：45 工作日 + 延长 15 工作日
└── 有效期：2 年

路径二：个人信息保护认证（一般个人信息）
├── 适用：一般个人信息出境
├── 流程：申请→认证→获证
├── 时限：30-60 工作日
└── 有效期：3 年

路径三：标准合同（小规模个人信息）
├── 适用：100 万以下个人信息、非重要数据
├── 流程：签署合同→备案
├── 时限：10 工作日备案
└── 有效期：合同期限
```

### 6.3 出境材料清单

**安全评估材料**：
1. 申报书
2. 数据出境风险自评估报告
3. 数据处理者与境外接收方拟订立的合同
4. 其他材料（业务说明、数据清单等）

**标准合同材料**：
1. 标准合同文本
2. 个人信息保护影响评估报告
3. 备案表

## 八、应急响应机制

### 8.1 安全事件分级

| 级别 | 定义 | 示例 | 响应时限 |
|------|------|------|----------|
| **特别重大 (I 级)** | 影响国家安全、大规模数据泄露 | 1000 万 + 个人信息泄露 | 1 小时内报告 |
| **重大 (II 级)** | 严重影响、大规模服务中断 | 100 万 -1000 万信息泄露 | 2 小时内报告 |
| **较大 (III 级)** | 较大影响、局部服务中断 | 10 万 -100 万信息泄露 | 4 小时内报告 |
| **一般 (IV 级)** | 一般影响、可快速恢复 | 10 万以下信息泄露 | 24 小时内报告 |

### 8.2 响应流程

```
【安全事件响应流程】

发现 → 报告 → 评估 → 处置 → 恢复 → 总结
 │      │      │      │      │      │
 │      │      │      │      │      └── 事件报告
 │      │      │      │      │          改进措施
 │      │      │      │      │
 │      │      │      │      └── 系统恢复
 │      │      │      │          业务恢复
 │      │      │      │          验证测试
 │      │      │      │
 │      │      │      └── 遏制扩散
 │      │      │          消除隐患
 │      │      │          证据保全
 │      │      │
 │      │      └── 事件定级
 │      │          影响评估
 │      │          方案制定
 │      │
 │      └── 初步报告
 │          启动预案
 │
 └── 监控发现
     用户报告
     外部通报
```

### 8.3 报告要求

| 报告对象 | 报告时限 | 报告内容 |
|----------|----------|----------|
| 内部管理层 | 立即 | 事件概况、影响范围 |
| 监管部门 | 按级别（见 8.1） | 详细报告、处置方案 |
| 受影响用户 | 72 小时内 | 事件说明、防护措施 |
| 社会公众 | 视情况 | 公告说明、应对措施 |
