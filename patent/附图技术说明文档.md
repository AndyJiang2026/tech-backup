## 图 1：系统整体架构图

### 技术说明

本图展示本发明系统的五大核心功能模块及其数据流向关系。

**模块说明：**

1. **数据采集模块**
   - 功能：实时采集用户在 VR 大空间中的位置坐标信息 (x, y, z) 和运动轨迹数据
   - 硬件组成：位置传感器、惯性测量单元（IMU）、深度摄像头
   - 采集频率：60Hz 至 240Hz

2. **空间分区模块**
   - 功能：根据用户位置将虚拟场景划分为不同优先级的渲染区域
   - 分区类型：核心体验区、过渡区、背景区
   - 分区依据：用户到场景中心的距离

3. **参数调整模块**
   - 功能：动态计算和调整渲染参数
   - 调整参数：LOD（细节层次）参数、空间扭曲系数 k
   - 调整策略：基于区域优先级和用户行为预测

4. **空间扭曲引擎**
   - 功能：对虚拟场景进行非线性映射变换
   - 核心算法：P'=P×(1-k×g(r))
   - 输出：适配用户实际活动范围的渲染画面

5. **输出模块**
   - 功能：将优化后的渲染画面输出至显示设备
   - 输出设备：VR 头显、外接显示器
   - 输出帧率：60-90 FPS

**数据流向：**
```
数据采集 → 空间分区 → 参数调整 → 空间扭曲引擎 → 输出
     ↑                                        ↓
     └──────────── 实时反馈回路 ──────────────┘
```

### 专利要点

- 模块化设计，各模块通过标准化接口进行数据交换
- 支持实时反馈回路，确保系统动态响应能力
- 核心算法公式在图中明确标注，便于审查员理解技术方案

## 图 3：动态参数调整流程图

### 技术说明

本图展示系统动态参数调整的完整处理流程，包含五个步骤和实时反馈回路。

**流程步骤：**

**S1：实时监测**
- 输入：用户位置坐标、视线方向、运动速度
- 处理：数据采集模块以 60-240Hz 频率采集原始数据
- 输出：标准化位置/姿态数据流

**S2：计算区域**
- 输入：标准化位置数据
- 处理：计算用户所在区域及优先级权重
- 输出：区域标识、优先级权重值

**S3：动态分配**
- 输入：区域优先级权重
- 处理：根据权重分配渲染资源（GPU 时间、显存）
- 输出：各区域渲染参数配置

**S4：应用空间扭曲参数**
- 输入：渲染参数配置
- 处理：计算空间扭曲系数 k，应用非线性映射变换
- 输出：变换后的场景几何数据

**S5：输出优化渲染帧**
- 输入：变换后的场景数据
- 处理：光栅化、后处理、显示输出
- 输出：最终渲染画面（60-90 FPS）

**反馈回路：**
- 从 S5 输出端实时监测帧率表现
- 根据帧率反馈动态调整 S3 的资源分配策略
- 确保系统稳定运行在目标帧率范围

### 专利要点

- 五步流程形成完整闭环
- 实时反馈机制确保系统稳定性
- 各步骤功能定义清晰，便于审查员理解

## 图 4：空间扭曲原理图

### 技术说明

本图通过对比方式展示空间扭曲算法的核心原理。

**变换公式：**

```
P' = P × (1 - k × g(r))
```

其中：
- P：原始顶点坐标
- P'：变换后顶点坐标
- k：空间扭曲系数，k = f(d, D)
- g(r)：距离衰减函数，g(r) = exp(-r²/2σ²)
- r：顶点到场景中心的距离
- σ：衰减半径参数（场景半径的 1/3 至 1/2）

**空间扭曲系数 k 的计算：**

```
当 d > D 时：        k = 0
当 D/4 < d ≤ D 时：   k = 0.1 × (D-d)/(3D/4)
当 d ≤ D/4 时：      k = 0.3 + 0.4 × (D/4-d)/(D/4)
```

其中：
- d：用户当前位置到最近安全边界的距离
- D：预设的预警距离阈值

**技术效果：**

- 当用户远离边界时（d > D），k = 0，无扭曲
- 当用户接近边界时（d ≤ D），k 逐渐增大，场景向中心收缩
- 最大扭曲系数 k_max = 0.7，确保视觉可接受性

### 专利要点

- 左侧原始网格与右侧扭曲网格形成直观对比
- 核心公式完整标注，便于审查员理解算法
- 变换效果清晰可见，符合专利充分公开要求

## 图 5：多用户协同时序图

### 技术说明

本图展示多用户协同 VR 体验中的状态同步时序关系。

**参与者：**

1. **主用户**
   - 角色：状态更新发起者
   - 帧率：60-90 FPS
   - 带宽：高优先级

2. **从用户 1、从用户 2**
   - 角色：状态接收者
   - 帧率：30-60 FPS
   - 带宽：标准优先级

3. **服务器**
   - 角色：状态同步中心
   - 功能：接收主用户状态，广播至所有从用户
   - 同步频率：30Hz

**同步流程：**

1. 主用户以 30Hz 频率向服务器发送状态数据（位置、姿态、动作）
2. 服务器进行状态融合和冲突解决
3. 服务器以 30Hz 频率广播同步状态至所有从用户
4. 从用户根据同步状态调整本地场景

**技术效果：**

- 确保多用户在同一虚拟场景中的一致性体验
- 主用户获得更高帧率和更低延迟
- 从用户通过插值算法平滑状态更新
